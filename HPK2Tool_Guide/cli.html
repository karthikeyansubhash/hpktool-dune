CLI Guide

HPK2Tool Command-Line Interface — Complete Reference

v2.0-beta10.0 HPK2Tool-cli.exe / HPK2ToolCli-linux.sh

Overview

The HPK2Tool CLI provides 10 commands covering the full HP Workpath solution lifecycle — from building a solution bundle out of an APK, to installing, configuring, and updating attestation data on a live device.

CommandPurpose
--createBuild a solution bundle from an APK and command-line options
--jsonBuild a solution bundle from an APK and a JSON manifest file
--installInstall a solution bundle onto a device
--uninstallUninstall a solution from a device
--solution-listList all installed solutions on a device
--solution-detailGet details for a specific installed solution
--config-getRetrieve solution configuration data from a device
--config-updatePush solution configuration data to a device
--attestation-updateUpdate HP.io attestation credentials on a device
--helpPrint help text
📌
One Command Per Invocation Only one command flag may be supplied per invocation. Combining two command flags (e.g. --create --install) will result in an error.

Running the CLI

.\HPK2Tool-cli.exe <command> [options]
HPK2Tool-cli.exe <command> [options]
./HPK2ToolCli-linux.sh <command> [options]
⚠️
JSON String Quoting Rule When passing JSON values (e.g. --application-title-main), always wrap the argument in double quotes.

If the value has no spaces:
"{"en-US":"MyTitle"}"

If the value contains spaces, wrap the value with an extra pair of double-quotes:
"{"en-US":""My Title""}"

--create — Build Solution Bundle

Packages an APK file and solution metadata into a deployable .hpk2 solution bundle. Optionally generates an .hpk file in the same run.

HPK2Tool-cli.exe --create --output <dir> --solution-id <uuid> --solution-name <name> ... --apk <file.apk> --platform-version <version>

OptionReqDescription
--outputRequiredOutput directory where the bundle will be written.
--solution-idRequiredUnique solution UUID (e.g. aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee).
--solution-nameRequiredSolution name. Used as the bundle filename.

--solution-phoneOptionalDeveloper support phone number.
--solution-urlOptionalDeveloper support URL.
--default-configOptionalPath to a default configuration JSON file embedded in the bundle.
--solution-bundle-serialOptionalBundle serial number string.

Application Options

Each solution supports one main application and up to 5 sub-applications (-1 through -5). Replace -main with -1, -2, etc. for sub-apps.

OptionReqDescription
--application-id-mainOptionalApplication UUID.
--application-name-mainRequired if id setApplication name.
--application-path-mainRequired if id setAndroid component path: package/ClassName.
--application-title-mainRequired if id setLocalized title JSON: {"en-US":"Title"}. See Localization Format.
--application-description-mainRequired if id setLocalized description JSON.
--application-icon-mainRequired if id setPath to an icon file (.png, .jpg, .jpeg).
--application-icon-set-mainRequired if id setIcon set JSON: {"90x90":"/path/icon.png"}. Overrides --application-icon-main.
--application-homeOptional flagRegister the main app as a Home Screen application.
--application-home-defaultOptional flagRegister as the default Home Screen application.

Authentication Agent Options

--authentication-nameRequired if id setAuthentication agent name.

--json — Build from JSON Manifest

Builds a solution bundle from an APK and a pre-written JSON manifest file, instead of individual CLI options. Useful for repeatable builds and CI pipelines.

HPK2Tool-cli.exe --json --output <dir> --apk <file.apk> --manifest <manifest.json>
OptionReqDescription
--outputRequiredOutput directory.
--apkRequiredPath to the solution .apk file.
--manifestRequiredPath to the JSON manifest file describing the solution.
--default-configOptionalPath to a default configuration JSON file to embed.

Example

.\HPK2Tool-cli.exe --json `
  --output "C:\output" `
  --apk "C:\apps\scanner.apk" `
  --manifest "C:\manifest\solution.json"
./HPK2ToolCli-linux.sh --json \
  --output "/home/user/output" \
  --apk "/home/user/apps/scanner.apk" \
  --manifest "/home/user/manifest/solution.json"

Installs a .bdl or .hpk2 bundle onto an HP Workpath device over the network.

HPK2Tool-cli.exe --install --install-file <bundle> --host <ip> [--password <pwd>]

--install-fileRequiredPath to the solution bundle (.bdl or .hpk2).
--hostRequiredDevice IP address.
--passwordOptionalDevice admin password. Omit if no password is set.

  --host "192.168.1.100" \

Localization JSON Format

Options such as --application-title-main, --authentication-title, and --webservice-title accept a JSON object mapping IETF language tags to localized strings. 

{
  "en-US": "Scan to Email",
  "ko-KR": "이메일로 스캔",
  "fr-FR": "Numériser vers E-mail",
  "de-DE": "An E-Mail scannen",
  "ja-JP": "メールにスキャン",
  "zh-CN": "扫描至电子邮件"
}
💡
Minimum Requirement At least one locale entry is required. en-US is used as the fallback locale when the device locale is not present in the JSON.

USB Accessory Format

Each --accessory-N value is a comma-separated string:

type,vendor-id,product-id[,serial-number]
FieldValuesDescription
typeOWNED or SHAREDOWNED — exclusive access; SHARED — shared with other solutions.
vendor-idHex (e.g. 0x04F9)USB Vendor ID.
product-idHex (e.g. 0x0027)USB Product ID.
serial-numberString or NULLOptional. Match a specific device by serial number.
 
# Without serial number
--accessory-1 "OWNED,0x04F9,0x0027"

# With serial number
--accessory-2 "SHARED,0x04F9,0x0027,SN12345678"

# Explicitly null serial number
--accessory-3 "OWNED,0x04F9,0x0027,NULL"

Web Service Endpoint Format

The --webservice-endpoint value uses a comma-separated format. Multiple endpoints are separated by a colon (:).

method,category,absolutePath,authType[:method,category,absolutePath,authType …]
FieldValuesDescription
methodGET, POST, PUT, DELETEHTTP method.
categoryStringEndpoint category (e.g. printing, scanning).
absolutePathStringAbsolute URL path (e.g. /api/v1/scan/jobs).
authTypeNONE, X_AUTH, ADMINAuthentication type required for this endpoint.
 
# Single endpoint
--webservice-endpoint "GET,scanning,/api/v1/scan/status,NONE"

# Multiple endpoints (colon-separated)
--webservice-endpoint "GET,scanning,/api/v1/scan/status,NONE:POST,scanning,/api/v1/scan/jobs,X_AUTH"

  --solution-version "1.0.0" `

  --platform-version 31.8 `
  --scan-agent
./HPK2ToolCli-linux.sh --create \
